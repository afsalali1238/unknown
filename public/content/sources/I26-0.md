Chinese AI lab Moonshot AI announced Kimi K3 on July 16, 2026, describing it as their "most capable model to date, with 2.8 trillion parameters." It's the first "open 3T-class model," taking the crown from DeepSeek's 1.6T V4 Pro. Self-reported benchmarks show K3 mostly beating Claude Opus 4.8 max and GPT-5.5 high, while losing out to Claude Fable 5 and GPT-5.6 Sol. Per an Artificial Analysis report: on a private long-horizon knowledge work evaluation, K3 reaches an overall Elo of 1547, +732 points from Kimi K2.6 and behind only Claude Fable 5; cost per task ($0.94) is similar to GPT-5.6 Sol ($1.04), about half the price of Opus 4.8 ($1.80); token usage on the Artificial Analysis Intelligence Index dropped 21% versus K2.6. K3 is also now the leading model on Arena.ai's Frontend Code arena, surpassing Claude Fable 5. Pricing is $3/million input and $15/million output tokens — matching Anthropic's Sonnet series and making K3 the most expensive model a Chinese lab has released to date, a sharp jump from Kimi K2.6's $0.95/$4.

**The pelican test.** Willison's running benchmark — "Generate an SVG of a pelican riding a bicycle" — is 21 months old as of this piece. It began as a joke about how hard it is to compare LLMs, then, for its first year, showed a genuine correlation with real model quality. Running it against Kimi K3 (via OpenRouter) cost 25 cents and consumed 16,658 output tokens, 13,241 of them reasoning tokens, because K3 currently ships with only one thinking-effort level: "max."

**Willison's own verdict on his benchmark:** "That connection has been mostly severed now. The GPT-5.6 and Claude Fable 5 pelicans are outclassed by GLM-5.2, and much as I love GLM I don't think that's a Fable-class model... So don't go using pelicans to compare models!" He notes he isn't convinced labs are directly training for the benchmark — if they were, he'd expect much better results — though he speculates Gemini "has optimized for any combination of an animal on a vehicle." The test's biggest limitation, in his words, is that it "doesn't touch at all on the thing that matters most for today's model: agentic tool calling and the ability to operate tools reliably as conversations grow in length."

Despite the decoupling, Willison says he still gets real value from running it: it forces him to actually try each new model; it gives a rough cost/reasoning estimate for a simple task; it confirms a model can output valid SVG with basic geometric and spatial awareness (more meaningful for small models running locally); it lets him compare pelicans across releases within the same model family as a rough capability-delta signal; and it's become a recognizable artifact he can share, to the point that Hacker News commenters now ask where the pelican is if his write-up runs late.
