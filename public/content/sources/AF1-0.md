> Archived 2026-07-17 for personal offline reading. All rights remain with the original authors.

Robert Metcalfe — co-inventor of Ethernet and founder of 3Com — first sketched this idea around 1980, and formally presented it to 3Com's sales force in 1983, not as an abstract law but as a sales argument for why customers should buy into networking early. It states that a network's value is proportional to the square of the number of connected, compatible devices on it (n²). George Gilder gave it the name "Metcalfe's Law" in a September 1993 Forbes article, applying it to the emerging internet.

## The math underneath

If a network has n nodes, the number of possible unique connections between them is n(n−1)/2 — a triangular number that grows asymptotically as n². Metcalfe's original, more careful formulation actually separated a linear cost term (Cn) from the non-linear value term (An²), where A ("affinity") represents value per connection — and he explicitly noted A likely declines as networks scale, since not every added user is equally valuable to every existing one (Metcalfe himself revisited and refined this in a 2006 interview).

## The classic illustration

A single fax machine is worthless. The second fax machine gives the first one something to talk to. Each additional machine doesn't just add one more possible connection — it adds one more connection to every existing machine simultaneously, so value compounds much faster than the raw count of participants.

## Contested but still standing

For 30+ years the law had no rigorous empirical backing, until a 2013 Dutch study found the n² relationship held for smaller networks (with an n·log(n) pattern showing up at larger scale), and Metcalfe himself later validated it further. In 2015, researchers confirmed the pattern held separately for both Facebook and Tencent data despite very different user bases. It has since been extended into finance — over 70% of the variance in Bitcoin's historical value has been linked to Metcalfe's law applied to its growing network size — and even into how mathematician Terence Tao has described why bigger research communities produce disproportionately better outcomes: more connections, not just more people.
